# jcode residual allowlist

> After rebrand cutover, residual `jcode` / `JCODE` / `Jcode` hits are **legal** only if they match one of the categories below.  
> Enforced by `scripts/rebrand/rg_gate.py`.  
> Contract: [`REBRAND_CONTRACT.md`](./REBRAND_CONTRACT.md).

Patterns are matched as **substrings or regexes** against a hit line (and sometimes path).  
When adding entries, prefer the narrowest pattern that covers the intended residual.

---

## 1. Path exclusions (whole file / tree skipped)

| Path pattern | Reason |
|---|---|
| `.git/**` | VCS internals |
| `target/**` | Build artifacts |
| `Cargo.lock` | Regenerated by cargo; not hand-edited |
| `changelog/**` | Historical release notes |
| `CHANGELOG*` | Historical release notes |
| `docs/*_PLAN.md` | Historical planning docs |
| `docs/plans/**` | Historical planning docs |
| `docs/REBRAND_*` | Rebrand audit/plan/contract/allowlist themselves |
| `docs/REBRAND_JCODE_TO_NEXT_CODE_AUDIT.md` | Audit of old brand |
| `docs/REBRAND_IMPLEMENTATION_PLAN.md` | Implementation plan |
| `docs/REBRAND_CONTRACT.md` | This contract set |
| `docs/REBRAND_ALLOWLIST.md` | This file |
| `**/*.{png,jpg,jpeg,gif,webp,ico,icns,pdf,woff,woff2,ttf,otf,bin,wasm,sqlite,db}` | Binary / non-text assets |
| `**/*.lock` | Lockfiles |

Historical image names (e.g. `docs/jcode_reddit_dashboard.png`) may keep filenames until a later asset pass; gate skips binary extensions.

## 2. Compat dual-read (intentional residuals)

These remain until the **compat removal** version (contract §2.5):

| Pattern / example | Reason |
|---|---|
| `JCODE_` dual-read env aliases | Env migration |
| `~/.jcode` / `.jcode/` dual-read paths | Storage migration |
| `jcode-provider-service` dual keyring read | Credential migration |
| `jcode-secrets` dual keyring read | Credential migration |
| `jcode://` dual URL scheme registration | Deep-link migration |
| `com.jcode.mobile` | **App Store bundle id — keep forever until explicit App Store rename** |
| Binary alias comments mentioning `jcode` | Install symlink docs |
| `legacy_jcode` / `old_jcode` / `jcode_compat` helper names | Explicit shim symbols |
| Comments: `// compat: jcode`, `// dual-read jcode` | Documented shims |

Gate: lines containing `dual-read`, `compat:`, `legacy_jcode`, `jcode_compat`, or `com.jcode.mobile` are allowlisted.

---

## 3. Third-party User-Agents and OAuth fingerprints

Must **not** be rewritten — they impersonate or match external products:

| Pattern | Reason |
|---|---|
| `claude-cli` | Anthropic/Claude CLI UA |
| `codex_cli_rs` | OpenAI Codex CLI fingerprint |
| `ChatGPT` / `OpenAI` product UA strings | Provider compatibility |
| Antigravity / Gemini browser UA strings | Provider compatibility |
| `DO_NOT_TRACK` | Standard privacy env (not brand) |

## 5. Telemetry / infra historical names

| Pattern | Reason |
|---|---|
| Cloudflare D1 `database_name` values containing `jcode` if kept historical | Infra rename is a separate migration |
| Wrangler / worker names documented as historical | Pending infra rename |
| `telemetry.jcode.sh` / `api.jcode.sh` / `jcode.sh` | **Domain freeze** (contract §2.3) |

Gate: lines matching `*.jcode.sh` or `jcode\.sh` are allowlisted until DNS decision.

## 6. Historical / narrative references

| Pattern | Reason |
|---|---|
| `1jehuang/jcode` in changelog, origin-sync notes, historical prose | Upstream history |
| Origin-sync skill logs mentioning upstream `jcode` | Process docs |
| Commit messages / PR titles quoted in docs | History |
| “formerly jcode” / “renamed from jcode” | Intentional narrative |

Live installers must use `quangdang46/next-code` — only **historical** paths keep `1jehuang/jcode`.

## 10. Maintenance rules

1. **Prefer fixing the product string** over expanding this allowlist.
2. New dual-read shims must reference this file and the removal TODO version.
3. After compat removal (P9), delete dual-read allowlist entries and re-run the gate green.
4. Domain allowlist entries drop when DNS migrates off `*.jcode.sh`.
5. Bundle id allowlist drops only after a deliberate App Store bundle migration.
